Getting Started

If you have never used git before, you need to download git first. Installation instructions for some common Operating Systems are given below.

Linux

To Install git on Linux, open a new terminal and run the following command.

              
                $ sudo apt-get install git

MacOS

It is highly recommended that you download homebrew package manager for a mac. Detailed instructions can be found here. With Homebrew installed, simply run the following command.

              
                $ brew install git

Windows

There are several ways to install git on windows. I will mention 2 popular ways here. Note that it is highly recommended that you use a *nix OS for this course.

Downloading git Directly

You can download git for windows directly from here. Follow on-screen instructions to install.

Installing Using Chocolatey Package Manner

Detailed instructions to install chocolatey (a package manager for windows) are here. With chocolatey installed, simply run the following command on the command line.

                
                  choco install git

Configuration

If you have never used git before, we need to do some basic one time config.

For *nix Users

Run the following commands from a terminal.

              
                $ git config --global user.name "Your Name"
                $ git config --global user.email "your_email@whatever.com"
                $ git config --global core.autocrlf input
                $ git config --global core.safecrlf true

For Windows Users

Run the following commands from the command line.

              
                $ git config --global user.name "Your Name"
                $ git config --global user.email "your_email@whatever.com"
                $ git config --global core.autocrlf true
                $ git config --global core.safecrlf true

Creating a git Repository

Let us start off by creating a new git repository.

First, let us create a new directory called  cs1217 .

              
                $ mkdir cs1217

Let us now  cd  into this directory.

              
                $ cd cs1217

To initialize an empty git repository, run the following command.

            
              cs1217$ git init

We now have an empty git repository in this folder.

Note that you do not need to initialize a repository if you clone a repository from GitHub. More on this later.

Starting a Project

Let us create a new file,  hello.c . To do this, run the following command.

              
                cs1217$ touch hello.c

The contents of  hello.c  should be the following.

              
                #include <stdio.h>
                
                int main(void)
                {
                   printf("Hello, World!\n");
                }

Now save this file, and type this in your terminal.

              
                cs1217$ git status

You will see  hello.c  in the untracked files.

Let us now go ahead and add  hello.c  to the staging area, so that it is ready to be committed.

              
                cs1217$ git add hello.c

Let us check the status now.

              
                cs1217$ git status

You should now see  hello.c  in the staging area, ready to be committed.

Let us go ahead and commit it.

              
                cs1217$ git commit -m "create hello.c"

The -m flag is used here to specify your commit message.

Commit messages should always be crisp, to the point, and easy to understand for your collaborators. They should ideally start with a verb.

If your commit message is longer than a 5-10 word sentence, chances are that you have not been committing properly and need to commit more frequently.

Now that we have done our first commit, it is time to push it to the repository. But before that, let us check the status yet again.

              
                cs1217$ git status
                On branch master
                nothing to commit, working tree clean

Notice that our working tree is clean now.

In order to push our changes to the repository, we must create a new repository on github.

Let us go to github and create a new private repository. To do so, click on the New button next to the repositories section on your homepage.

Set repository name to cs1217 and select the private option. Ignore everything else for now, and click create repository.

We now need to link our local repository to this remote one. In order to do this, let us first copy the link of our newly created repository. It should look something like this

              
                https://github.com/checker5965/cs1217.git

Let us go back to our terminal and execute the following commands.

              
                cs1217$ git remote add origin https://github.com/checker5965/cs1217.git
                cs1217$ git push -u origin master

And with that, we have just pushed our changes to our remote repository. You can go to github and check the repository out to see that your changes are there.

Working on an Existing Project

Let us first clean our workspace. Run the following commands.

              
                cs1217$ cd ..
                $ rm -rf cs1217

To download an existing repository from Github, we use the command  git clone [link to repo]. For this tutorial, we will redownload the repository we created earlier. To do this, run the following command.

              
                $ git clone https://github.com/checker5965/cs1217.git

Let us  cd  into the directory and  ls  to make sure  hello.c  is in the directory.

In order to make changes to the repository now, we can use the previous workflow without the need to add origin again.

Branching and Merging

In a collaborative environment, developers often want to work on different features and files simultaneously. You may choose to divide your work in the labs for this course, and thus, knowing how branches work could be an important thing.

Let us now create a new branch called  myhello . To do this, run the following command.

              
                cs1217$ git checkout -b myhello

On checking the status, the output should be

              
                cs1217$ git status
                On branch myhello
                nothing to commit, working tree clean

Let us now change  hello.c  such that it accepts the user's name and prints  Hello, [username]!  The contents should be the following.

              
                #include <stdio.h>
                #include <assert.h>
                  
                int main(int argc, char** argv)
                {
                     assert(argc==2 && "Only one argument allowed!");
                     printf("Hello, %s!\n", argv[1]);
                }

On checking the status, we will see that  hello.c  is listed as modified. Let us push this change.

              
                cs1217$ git add hello.c
                cs1217$ git commit -m "add ability to print username in hello.c"
                cs1217$ git push origin myhello

You can now go to github and see that your  master  branch will not have the changes, while your  myhello  branch will.

Let us switch to master, and merge our changes.

              
                cs1217$ git checkout master
                cs1217$ git merge myhello

Remember that our master branch is now one commit ahead of the remote repository, and these changes need to be pushed.

              
                cs1217$ git push

Important note - you do not need to know advanced git in order to work with the code for this course, however, be very careful while working with git. As a rule of thumb, it might be best for the two partners to work on two separate branches (neither of which is master) and on different files.

Undoing Changes

Suppose you made a mistake while adding a new feature to  hello.c  

              
                #include <stdio.h>
                #include <assert.h>
                  
                int main(int argc, char** argv)
                {
                  // Assert command is checking the wrong condition.
                     assert(argc==1 && "Only one argument allowed!");
                     printf("Hello, %s!\n", argv[1]);
                }

If you have not staged your change yet (i.e. you have not added the change via  git add  ) you can simply use the command below to revert to the latest commit.

              
                cs1217$ git checkout hello.c

Suppose you did not realize that you made an error and already added it to the staging area via  git add  , use the command below to unstage the commit.

              
                cs1217$ git reset HEAD hello.c

And now, you can simply use the checkout command to revert the changes locally.

              
                cs1217$ git checkout hello.c

Now, suppose you have already commited the change (but not pushed it), you can use the command below to undo the commit.

              
                cs1217$ git revert HEAD

Note that this has put your current branch two commits ahead of the remote branch. Using revert does NOT delete the prior commit. Instead, it creates a new commit with the code in a previous commit.

Another important thing to note here is that we could revert using HEAD only because we wanted to go one commit back. Another way to revert would be to specify the hash of a commit instead of HEAD. You can find the hash of commits using the  git log  command.
